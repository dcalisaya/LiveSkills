# Pre-Production Reference — Live Developer Stack

## Brief Parsing

Every video project starts with a brief. Parse it into this structure:

### Brief Template

```
## Video Brief

- **Project:** [Name]
- **Client:** [Who]
- **Format:** [Reel / Explainer / Testimonial / Ad / Brand Film]
- **Duration:** [Target length]
- **Platform:** [YouTube, Instagram, Website, LinkedIn, etc.]
- **Audience:** [Who watches? What do they know? What do they feel?]
- **Core message:** [ONE sentence — what should viewers remember?]
- **CTA:** [What should viewers DO after watching?]
- **Tone:** [Professional / Casual / Cinematic / Energetic / Emotional]
- **Assets available:** [Logos, footage, photos, music, brand guidelines]
- **Deadline:** [Delivery date]
- **Budget notes:** [Constraints, talent availability, equipment]
```

### Brief Quality Checklist

```
□ Core message is one clear sentence
□ Audience is specific, not "everyone"
□ Format matches the platform
□ Duration is realistic for the content
□ CTA is concrete and measurable
□ Assets list is complete
□ Deadline allows for at least one review cycle
```

## Script Writing

### Script Format (Two-Column)

```
TIMECODE | VISUAL                          | AUDIO / NARRATION
---------|----------------------------------|----------------------------------
00:00    | Black screen → Logo reveal       | [Music: ambient build]
00:03    | Wide shot: agency office          | "Managing deliverables for 10
         | People working, screens visible  |  clients shouldn't feel like chaos."
00:08    | CU: laptop showing overflowing   | "But for most agencies, it does."
         | email inbox                      |
00:12    | Screen recording: LiveApp        | "LiveApp automates the workflow —
         | dashboard, smooth scroll         |  from brief to delivery."
00:18    | Split screen: before (messy) vs  | "So your team focuses on the work
         | after (clean dashboard)          |  that matters."
00:23    | Logo + CTA text                  | "Start free at livedeveloper.com"
         |                                  | [Music: resolve]
```

### Script Writing Rules

- **Hook in 0-3 seconds** — Open with a question, bold statement, or visual surprise.
- **One idea per scene** — Don't stack concepts. Let each one breathe.
- **Write for the ear** — Read it out loud. If it sounds weird spoken, rewrite it.
- **Visual-first** — If you can show it, don't say it. Narration supports visuals.
- **End with CTA** — Last 3-5 seconds: what to do next. Clear, simple, on screen.
- **Brevity** — Every word must earn its place. Cut ruthlessly.

### Duration Guide (Words to Time)

| Pace | Words per Minute | 30s | 60s | 120s |
|---|---|---|---|---|
| Slow (cinematic) | 100-120 | 50-60 words | 100-120 | 200-240 |
| Normal (explainer) | 140-160 | 70-80 words | 140-160 | 280-320 |
| Fast (social) | 170-200 | 85-100 words | 170-200 | 340-400 |

## Storyboard Creation

### Storyboard Panel Structure

Each panel includes:

```
## Panel [Number] — [Scene Title]

**Timecode:** 00:XX - 00:XX
**Shot type:** [Wide / Medium / Close-up / CU Detail / OTS / POV / Aerial]
**Camera movement:** [Static / Pan / Tilt / Dolly / Handheld / Drone]
**Framing:** [Rule of thirds / Center / Symmetrical / Leading lines]
**Visual description:** [What the viewer sees]
**On-screen text:** [Any text, titles, or lower thirds]
**Audio:** [Narration, dialogue, music, SFX]
**Transition to next:** [Cut / Dissolve / Wipe / Match cut / J-cut]
**Mood/lighting:** [Bright / Dark / Warm / Cool / High contrast]
```

### Shot Type Quick Reference

| Shot | Abbreviation | Use For |
|---|---|---|
| Extreme Wide | EWS | Establishing location, scale |
| Wide Shot | WS | Context, environment, full body |
| Medium Shot | MS | Waist up, conversational |
| Medium Close-Up | MCU | Chest up, interview framing |
| Close-Up | CU | Face, emotion, detail |
| Extreme Close-Up | ECU | Eyes, hands, product detail |
| Over-the-Shoulder | OTS | Dialogue, screen showing |
| Point of View | POV | Viewer sees through character's eyes |
| Bird's Eye / Aerial | AE | Overview, transitions, establishing |

## Shot List Template

```
## Shot List — [Project Name]

| # | Scene | Shot | Description | Equipment | Location | Talent | Duration | Notes |
|---|---|---|---|---|---|---|---|---|
| 1 | Opening | WS | Office exterior, morning light | Drone | HQ | None | 5s | Golden hour |
| 2 | Opening | MS | Team walking in, carrying coffee | Gimbal | Lobby | 3 extras | 4s | Casual energy |
| 3 | Problem | CU | Laptop screen with messy inbox | Tripod + macro | Desk | Hand model | 3s | Screen recording |
```

### Shot List Rules

- Order by **location**, not by timeline — minimize setup changes.
- Mark equipment needs per shot — camera, lens, lighting, audio.
- Include talent requirements — who needs to be on set for each shot.
- Note any props, wardrobe, or set dressing required.
- Add a "priority" column if time is tight (must-have vs nice-to-have).
