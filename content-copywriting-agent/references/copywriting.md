# Copywriting Reference — Live Developer Stack

## Core Principles

1. **Clarity wins.** If the reader has to re-read a sentence, rewrite it.
2. **Benefits over features.** Features describe what it does. Benefits describe why they should care.
3. **One CTA per context.** Don't ask the reader to do five things. Pick one.
4. **Specificity sells.** "Reduces onboarding from 2 weeks to 3 days" beats "saves time."
5. **Remove every word that doesn't earn its place.**

## Headline Formulas

### Proven Structures

| Formula | Example |
|---|---|
| **[Number] Ways to [Achieve Benefit]** | "7 Ways to Ship Client Deliverables Faster" |
| **How to [Do X] Without [Pain Point]** | "How to Scale Your Agency Without Hiring More PMs" |
| **[Do X] Like [Aspirational Reference]** | "Manage Projects Like a Top 1% Agency" |
| **The [Adjective] Guide to [Topic]** | "The Complete Guide to Agency Workflow Automation" |
| **Why [Common Practice] Is [Wrong/Outdated]** | "Why Spreadsheets Are Killing Your Agency's Margins" |
| **[Result] in [Timeframe]** | "3× Faster Deliverables in 30 Days" |
| **Stop [Pain]. Start [Benefit].** | "Stop Chasing Updates. Start Shipping Work." |

### Headline Quality Checklist

```
□ Contains a clear benefit or outcome
□ Speaks to a specific audience
□ Creates curiosity or urgency
□ Under 60 characters (SEO title) or under 10 words (ad headline)
□ No jargon the audience wouldn't use
□ Would you click on this yourself?
```

## Persuasion Frameworks

### PAS (Problem → Agitate → Solution)

Best for: landing pages, email opens, ad copy.

```
PROBLEM: "Managing deliverables across 10+ clients is chaos."
AGITATE: "You spend more time on status updates than actual work.
          Deadlines slip. Clients lose trust. Your best people burn out."
SOLUTION: "LiveApp automates the handoffs so your team focuses on
           what they were hired to do — create great work."
```

### AIDA (Attention → Interest → Desire → Action)

Best for: landing pages, long-form sales pages.

```
ATTENTION: "What if your agency could ship 3× faster?"
INTEREST:  "LiveApp uses AI agents to automate brief parsing,
            review cycles, and client communication."
DESIRE:    "120+ agencies already save 8 hours/week per PM.
            No more chasing approvals over email."
ACTION:    "Start your free trial → no credit card required."
```

### BAB (Before → After → Bridge)

Best for: email copy, case studies, social posts.

```
BEFORE: "Before LiveApp, a major B2B enterprise client managed deliverables via email
         threads and shared drives. Average review cycle: 9 days."
AFTER:  "After LiveApp, review cycles dropped to 2.5 days.
         Client satisfaction scores went from 7.1 to 9.3."
BRIDGE: "The bridge? AI-powered workflow automation that keeps
         everyone on the same page — literally."
```

## CTA Writing

### Rules

- **Start with an action verb**: Start, Get, Try, See, Book, Download
- **Promise an outcome**: "Start Free Trial" > "Submit" | "See It In Action" > "Learn More"
- **Max 4 words** for primary CTA
- **One primary CTA** per section. Ghost/secondary CTA is softer.

### CTA Examples by Context

| Context | Primary CTA | Secondary CTA |
|---|---|---|
| SaaS landing | "Start Free Trial" | "See Pricing" |
| Demo request | "Book a Demo" | "Watch the Video" |
| Blog post | "Read the Full Guide" | "Subscribe for Updates" |
| Email campaign | "Claim Your Spot" | "Learn More" |
| Pricing page | "Start Building" | "Compare Plans" |

## Landing Page Copy Structure

### Above the Fold

```
[Eyebrow] — Who this is for (e.g., "For Marketing Agencies")
[Headline] — One clear promise
[Subheadline] — Expand on the headline in 1-2 sentences
[CTA Button] — Primary action
[Social proof] — Logo bar or key metric
```

### Below the Fold

```
[Problem section] — Articulate the pain (2-3 short paragraphs)
[Solution section] — Introduce the product as the answer
[Features/Benefits] — 3-6 items, lead with benefit
[How it works] — 3-step visual process
[Testimonials] — Real quotes with names and roles
[FAQ] — 5-7 questions addressing objections
[Final CTA] — Repeat the primary CTA with urgency
```

## Tone Spectrum

Choose the right position for each project:

```
Formal ──────────────── Conversational
  │                           │
  ├─ Legal documents          ├─ Blog posts
  ├─ Enterprise proposals     ├─ Email campaigns
  ├─ Government clients       ├─ Social media
  └─ Annual reports           └─ Onboarding UX copy
```

### Live Developer Default: Professional Conversational

- Write like a smart colleague explaining something clearly.
- Use contractions ("we're", "you'll") but avoid slang.
- Use short sentences. Mix with the occasional longer one for rhythm.
- Direct address: "you" and "your" — it's about the reader.

## Microcopy Guidelines

Microcopy = the small text that guides users through an interface.

| Element | ❌ Don't | ✅ Do |
|---|---|---|
| Empty state | "No data" | "No deliverables yet. Create your first one." |
| Error message | "Error 500" | "Something went wrong. Try again, or contact support." |
| Loading | "Loading..." | "Pulling your latest projects..." |
| Success | "Done" | "Project created successfully. You can start adding deliverables." |
| Button label | "Submit" | "Create Project" |
| Tooltip | "Click here" | "Open project settings" |
| Placeholder | "Enter text" | "e.g., Brand Kit v2" |

## Writing Checklist (Before Publishing)

```
□ Read it out loud. Does it sound natural?
□ Cut 20% of the words. Is it tighter now?
□ Replace every adjective with evidence. ("fast" → "3× faster")
□ Check every sentence: does the reader care?
□ Is there ONE clear CTA?
□ Would a 12-year-old understand the main idea? (clarity test)
□ Does it pass the "so what?" test? (every paragraph must earn its place)
□ Spell check, grammar check, link check
```
